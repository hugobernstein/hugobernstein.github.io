intro

Experienced programmers, you will find this an easy read. Inexperienced programmers, you might find some parts a trudge, but keep going. Learning the material on this page will pay hefty dividends for as long as you use Pollen.

Tag functions

Three golden rules of tag functions:
  1. Every Pollen tag calls a function with the same name.
  2. The input values for that function are the attributes and elements of the tag.
  3. The whole tag—tag name, attributes, and elements—is replaced with the return value of the called function.

Tag-function syntax

Det finns tre typer av Tag-function: tag alone, tag med elements samt tag med attributes och elements.

Commands can be written in pollen mode or Racket mode

A tag without attributes or elements is interpreted as a value.

To define a tag as a function, square brackets is needed to signal that you want to evaluate the function.

to refer to the value held by a tag name: a tag holding a value or a tag holding a value to invoke the function: a tag function returning a value

Parsing attributes

detecting attributes in an argument list can be tricky, because:

  1. the tag may or may not have attributes;
  2. those attributes may be in standard or abbreviated syntax.
